The use case

Servers run out of disk space at different urgency levels. At 72% you do nothing. At 85% you reclaim cache and old logs before it hurts. At 97% you add capacity — on EC2, expand the EBS volume, grow the partition, and extend XFS. Each path posts a tier-specific Mattermost message so the team knows what ran.

Workflow

Check disk usage
       |
  Switch on disk_use_percent
       |
-------------------------------------------------
<80%     80-95%     >95%     (default)
  |         |          |          |
Continue   Cleanup   Expand    Fallback
  |         |          |          |
 Notify    Notify    Notify    Notify
 (ok)      (warn)   (critical) (unsupported)

What you'll see

Four switch ports, four small remediate playbooks, and four notify templates (green, orange, purple, red). Use test_disk_use_percent on the check job to demo each branch without filling the disk.
